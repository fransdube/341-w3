📚 Bookstore API Documentation

RESTful API for managing books with full CRUD operations

Base URL: https://your-render-app.onrender.com/api/books

🔍 Overview

This API provides endpoints to perform CRUD operations on a MongoDB database for a bookstore collection. All endpoints return JSON responses with proper HTTP status codes and error handling.

📋 Book Schema (7+ fields)

Field Type Required Validation
titleString✅1-200 characters
authorString✅2-100 characters
isbnString✅Valid ISBN (10/13 digits)
publishedYearNumber✅1450-current year
genreString✅Predefined genres only
priceNumber✅0-10000
pageCountNumber✅1-5000
publisherString✅Minimum 2 characters
inStockBoolean❌Default: true
ratingNumber❌0-5, default: 0

🚀 Endpoints

GET /api/books

Description: Retrieve all books with filtering, sorting, and pagination

Query Parameters (optional):

  • genre - Filter by genre
  • minPrice - Minimum price filter
  • maxPrice - Maximum price filter
  • inStock - Filter by availability (true/false)
  • sort - Sort field:order (e.g., price:desc)
  • page - Page number (default: 1)
  • limit - Items per page (default: 10)

Example Response:

{
  "success": true,
  "count": 5,
  "total": 25,
  "page": 1,
  "pages": 5,
  "data": [
    {
      "_id": "507f1f77bcf86cd799439011",
      "title": "The Great Gatsby",
      "author": "F. Scott Fitzgerald",
      "isbn": "9780743273565",
      "publishedYear": 1925,
      "genre": "Fiction",
      "price": 12.99,
      "inStock": true,
      "pageCount": 180,
      "publisher": "Scribner",
      "rating": 4.5
    }
  ]
}

Description: Retrieve a single book by its MongoDB ID

Example Response:

    "inStock": true,

    "publisher": "Scribner",

POST /api/books

Description: Create a new book

Request Body (required fields):

{
  "title": "To Kill a Mockingbird",
  "author": "Harper Lee",
  "isbn": "9780061120084",
  "publishedYear": 1960,
  "genre": "Fiction",
  "price": 14.99,
  "pageCount": 336,
  "publisher": "J.B. Lippincott & Co.",
  "inStock": true,
  "rating": 4.8
}

Example Response:

{
  "success": true,
  "message": "Book created successfully",
  "data": {
    "_id": "507f1f77bcf86cd799439012",
    "title": "To Kill a Mockingbird",
    ...
  }
}
PUT /api/books/:id

Description: Update an existing book (partial updates allowed)

Example Request Body:

{
  "price": 16.99,
  "inStock": false
}

Example Response:

{
  "success": true,
  "message": "Book updated successfully",
  "data": { ...updated book... }
}
DELETE /api/books/:id

Description: Delete a book by ID

Example Response:

{
  "success": true,
  "message": "Book deleted successfully",
  "data": {
    "id": "507f1f77bcf86cd799439011",
    "title": "The Great Gatsby"
  }
}
GET /api/books/search?q=keyword

Description: Search books by title, author, or genre

Example Response:

{
  "success": true,
  "count": 3,
  "data": [ ...matching books... ]
}

⚠️ Error Responses

400 Bad Request (Validation Error)

{
  "success": false,
  "message": "Validation failed",
  "errors": [
    {
      "field": "price",
      "message": "Price must be between 0 and 10000"
    }
  ]
}

404 Not Found

{
  "success": false,
  "message": "Book not found with id: 507f1f77bcf86cd799439999"
}

500 Server Error

{
  "success": false,
  "message": "Server Error"
}

📡 Testing with Postman

  1. Import the base URL: http://localhost:3000 (local) or your Render URL
  2. Test POST: Body → raw → JSON
  3. Test GET with query params: /api/books?genre=Fiction&minPrice=10&sort=price:desc
  4. Test PUT: Include book ID in URL
  5. Test DELETE: Include book ID in URL

🚀 Deployment on Render

  1. Push code to GitHub
  2. Create new Web Service on Render
  3. Connect GitHub repository
  4. Add environment variables:
    • MONGODB_URI = your MongoDB connection string
    • PORT = 3000
  5. Build command: npm install
  6. Start command: node server.js
